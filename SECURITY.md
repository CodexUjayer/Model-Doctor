# Security Policy

## Supported Versions

The latest stable release of ModelDoctor is supported with security updates.

| Version | Supported |
|----------|-----------|
| Latest Release | ✅ |
| Older Releases | ❌ |

---

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability in ModelDoctor, please **do not open a public GitHub Issue**.

Instead, report it privately by emailing:

dev.ujayerhasnat@gmail.com

Include as much information as possible:

- ModelDoctor version
- Python version
- Operating system
- Steps to reproduce
- Proof of concept (if applicable)
- Potential impact

We will:

- Acknowledge your report within 7 days.
- Investigate the issue.
- Keep you informed of progress.
- Credit you (if desired) after the issue is resolved.

---

## Scope

Examples include:

- Remote code execution
- Arbitrary code execution
- Unsafe deserialization
- Credential leakage
- Dependency vulnerabilities
- Injection vulnerabilities
- Privilege escalation

General bugs, feature requests, and documentation issues should be reported through GitHub Issues instead.

Thank you for helping keep ModelDoctor secure.
